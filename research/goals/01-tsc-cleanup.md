# Goal 01 — TypeScript error cleanup

## Metric

```bash
npx tsc --noEmit -p tsconfig.app.json 2>&1 | grep -c "error TS" || echo 0
```

Output is a single integer. Lower is better. Target is 0.

## Baseline (at spec authoring time)

48 errors, concentrated in:

- `src/pages/EditProfile.tsx`
- `src/pages/Onboarding.tsx`
- `src/pages/PublicCoachProfile.tsx`
- `src/hooks/v2/useSupabaseQueries.ts`

These are pre-existing in v1 code. They do NOT block `npm run build` because
Vite's build doesn't run `tsc` strictly — but they block any future migration
to stricter compiler settings and they hide real regressions.

## Scope

**In-scope files:**
- Everything under `src/`.
- Any migration / config file the compiler touches.

**Out-of-scope (do not modify):**
- `src/pages/v2/auth/**` — shipped, stable, passes tsc as-is.
- `src/pages/v2/BookLandingPage.tsx` + Book hooks — shipped.
- `ios/**`, `android/**` — native projects, auto-generated by Capacitor.
- `dist/**`, `node_modules/**`, `.git/**`.

## Anti-cheating rules

Agent must NOT use any of these to force the metric down:

- `@ts-ignore`
- `@ts-expect-error`
- `as any`
- `as unknown as X`
- Non-null assertion (`!`) introduced solely to silence errors
- `// eslint-disable` additions
- Changing `tsconfig*.json` to relax `strict`, `noImplicitAny`,
  `strictNullChecks`, etc.
- Deleting files purely because they have errors
- Replacing typed calls with untyped fallbacks (e.g. `fetch` instead of
  the Supabase client) to bypass type checks

If the agent finds a genuinely-broken type in a dependency (e.g. a Supabase
schema type that's stale), the correct fix is to:
1. Regenerate the types (`supabase gen types typescript`) — if possible
2. Narrow the call with a properly-typed cast reviewed by the coordinator
3. Skip the error and report it in the iteration log, explaining why

## Stop conditions

The coordinator will exit when ANY of these is true:

- Metric reaches 0
- Max iterations exhausted (default 20, overridable with `--max-iter`)
- Three consecutive iterations with zero metric improvement
- Agent emits the literal token `STUCK` in its output

## Success criterion

`tsc --noEmit -p tsconfig.app.json` exits with code 0 AND:

- `npm run build` still succeeds
- `npx eslint .` warning count has not grown
- `git diff main...HEAD` contains no file from the out-of-scope list above
- `git grep -E "(@ts-ignore|@ts-expect-error|as any|// eslint-disable)"` on
  the branch does not show any ADDITIONS compared to main

When all four are true the coordinator opens a PR titled
`chore(types): autonomous tsc cleanup` for human review.

## Prompt hint

When the coordinator calls the agent each iteration, it prepends this
problem statement to the prompt:

> You are fixing TypeScript errors in this repository. Run
> `npx tsc --noEmit -p tsconfig.app.json 2>&1` to see the current error
> list. Pick ONE error, understand its root cause, and fix it properly
> without using any of the anti-cheating patterns listed in
> [research/goals/01-tsc-cleanup.md](research/goals/01-tsc-cleanup.md).
> After your edit, verify by running tsc again. If your edit does not
> strictly decrease the error count, revert it yourself and try a
> different error. When you are done with this iteration, stop — do not
> attempt to fix multiple errors at once.
