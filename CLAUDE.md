## Commands

```bash
# Dev (same web experience, full hot reload)
npm run dev                 # http://localhost:8080

# Build web assets → dist/
npm run build

# Capacitor
npm run cap:sync            # build + sync web assets into ios/ and android/
npm run cap:copy            # copy dist/ into native projects (no plugin refresh)
npm run cap:ios             # build + sync ios + open Xcode
npm run cap:android         # build + sync android + open Android Studio
npm run cap:run:ios         # build + run on connected iOS device/simulator
npm run cap:run:android     # build + run on connected Android device/emulator

# Lint/tests
npm run lint
npm run test
```

### Live-reload on a physical device

```bash
CAP_SERVER_URL=http://<your-lan-ip>:8080 npm run cap:sync
npm run dev                 # leave running — the device webview loads from here
```

`CAP_SERVER_URL` is read by `capacitor.config.ts`. When unset, the app loads from
the bundled `dist/` assets (normal production mode).

## Native integration — how to use it in components

**Feature-detect the platform:**
```tsx
import { usePlatform } from "@/native/useNative";
const { isNative, platform } = usePlatform(); // "ios" | "android" | "web"
```

**Haptics on important taps:**
```tsx
import { useHaptics } from "@/native/useNative";
const { tap, success, error } = useHaptics();
<Button onClick={() => { tap("light"); bookSession(); }}>Book</Button>
```

**Native share sheet:**
```tsx
import { useShare } from "@/native/useNative";
const share = useShare();
share({ title: "Check out this coach", url: `https://circloclub.com/coach/${id}` });
```

**Secure key-value storage (replaces localStorage on native):**
```tsx
import { nativeStorage } from "@/native/useNative";
await nativeStorage.set("circlo_active_role", "coach");
const role = await nativeStorage.get("circlo_active_role");
```
⚠ `AuthContext` and `ThemeContext` still use `localStorage` directly — that's
fine for now (Capacitor's web runtime persists it), but migrate to `nativeStorage`
if you want values to survive across iOS app updates.

**Network status:**
```tsx
import { useNetworkStatus } from "@/native/useNative";
const { connected, connectionType } = useNetworkStatus();
```

## Safe-area / notch handling

`src/styles/native.css` exposes CSS vars + two class hooks:
- `.app-top-nav` → pads top by `env(safe-area-inset-top)` on native
- `.app-bottom-nav` → pads bottom by `env(safe-area-inset-bottom) + 8px`

Both are already applied in `AppShell.tsx`. When you build new full-bleed pages
that render their own chrome (e.g. `Reels`, `Plays`), add these classes to any
sticky/fixed elements that sit at the very top or bottom of the screen.

## Hard rules (inherited + mobile-specific)

All rules from the website's `CLAUDE.md` apply. On top of that:

1. **Never import `@capacitor/*` directly inside components** — always go through
   `src/native/useNative.ts`. This keeps the web build identical to the website
   and all native calls safely no-op on the browser.
2. **Never commit `dist/`**, `ios/App/build/`, `android/app/build/`, `Pods/`,
   `.gradle/`, `local.properties`. See `.gitignore`.
3. **Never change the bundle ID** (`club.circlo.app`) — it's tied to App Store /
   Play Store listings.
4. **Never ship without running `npm run cap:sync`** — the native projects load
   from `dist/`, so stale `dist/` = stale app.
5. **Run `npm run build` before any commit** — inherited hard rule.

## Prereqs for native builds

- **iOS:** Xcode 15+, CocoaPods (`sudo gem install cocoapods`). Open with
  `npm run cap:ios`.
- **Android:** Android Studio Hedgehog+, JDK 17, Android SDK 34. Open with
  `npm run cap:android`.
- **First-time iOS pod install:** after installing CocoaPods, run
  `cd ios/App && pod install` once.
